✅
Méthode recommandée

Ajouter le dépôt personnalisé

  • URL : https://github.com/JeanMarc-Labs/ha-config-auditor

Utilisation

Tableau de bord

La première page que vous voyez en ouvrant H.A.C.A. Vue d'ensemble instantanée de la santé de votre installation.

Score de santé

Le score (0–100) est affiché en grand en haut du tableau de bord. Il se met à jour après chaque scan. Le calcul :

formule 
Score = 100 − Σ pénalités

Pénalités par issue :
  HIGH   → −5 points
  MEDIUM → −2 points
  LOW    → −1 point

Statistiques rapides

Six compteurs sous le score permettent d'accéder directement à chaque catégorie :

Compteur Description
Issues totales Toutes catégories confondues
Automatisations Issues automations / scripts / scènes
Entités Entités problématiques détectées
Performance Problèmes de performance
Sécurité Vulnérabilités potentielles
Batteries Appareils sous le seuil d'alerte

Graphique d'évolution

Un graphique sparkline montre l'évolution du score sur les 30 derniers scans. Survolez les points pour voir le détail. Une hausse stable = vos corrections portent leurs fruits. Une baisse soudaine = nouvelles automations avec problèmes.

Filtrer les issues

L'interface propose plusieurs niveaux de filtrage :

  • Onglets thématiques — Automatisations, Scripts, Scènes, Entités, Performance, Sécurité, Blueprints
  • Chips de sévérité — HIGH MEDIUM LOW
  • Export CSV — Exporter la liste filtrée courante

Principaux types d'issues

device_id_in_trigger / device_id_in_action / device_id_in_target

🔴
Problème — Sévérité HIGH
Un déclencheur, action ou target utilise device_id au lieu d'une entité. Si le device est supprimé et réajouté (remplacement matériel), son ID change et l'automatisation casse silencieusement, sans aucune erreur visible.
yaml · diff 
# ❌ Avant (fragile)
- platform: device
  device_id: abc123def456
  domain: binary_sensor
  type: motion

# ✅ Après (robuste — corrigé par H.A.C.A)
- platform: state
  entity_id: binary_sensor.capteur_mouvement_salon
  to: "on"

incorrect_mode_motion_single

🔴
Problème — Sévérité HIGH
Mode single sur une automatisation mouvement/lumière. En mode single, si le capteur se déclenche pendant l'exécution (extinction programmée), l'action est ignorée — la lumière reste allumée indéfiniment.
yaml · diff 
mode: single
mode: restart   # ✅ Redémarre si nouveau mouvement détecté

template_simple_state

🟡
Problème — Sévérité MEDIUM
Un template complexe fait une vérification d'état simple qui pourrait être exprimée nativement. Les templates recalculent à chaque changement d'état dans HA, même quand ce n'est pas nécessaire.
yaml · diff 
# ❌ Avant (template inutilement complexe)
condition:
  - condition: template
    value_template: "{{ states('light.salon') == 'on' }}"

# ✅ Après (condition native)
condition:
  - condition: state
    entity_id: light.salon
    state: "on"

no_alias / no_description

🔵
Problème — Sévérité LOW
Automatisation ou script sans nom lisible ou description. Rend la maintenance difficile — dans 6 mois, vous ne saurez plus ce que fait cette automatisation.

H.A.C.A utilise l'IA pour suggérer un nom/description basé sur le contenu YAML. Le texte est modifiable avant application.

expensive_template_states_all

🔴
Problème — Sévérité HIGH
Utilisation de states (sans entité ciblée) dans un template. Cette expression parcourt TOUTES les entités HA à chaque évaluation. Sur une installation avec 500+ entités, cela crée une charge CPU significative.

Main issue types

🔴
HIGH — Breaks silently on device replacement
Using device_id instead of entity_id. When a device is removed and re-added (hardware replacement), its ID changes and the automation silently stops working.

🔴

single mode on a motion/light automation means re-triggers during execution are ignored. The light can stay on forever.
yaml · diff

Module Recorder DB

⚠️
Pourquoi c'est important
Les entités orphelines (enregistrées en DB mais inexistantes dans HA) gaspillent de l'espace disque et ralentissent les requêtes historiques. Sur de grandes installations, cela peut représenter plusieurs Go de données inutiles.

Purger les orphelins

🔴
Action irréversible
La purge supprime définitivement les données historiques de ces entités. Créez une sauvegarde HA complète avant.
yaml 
service: config_auditor.purge_recorder_orphans

Prévenir les futurs orphelins

yaml · configuration.yaml 
recorder:
  purge_keep_days: 30
  exclude:
    entity_globs:
      - sensor.time_*
      - sensor.date_*
    domains:
      - weather
      - sun

Explication des issues par l'IA

Le bouton 🤖 Expliquer génère pour chaque issue :

  • Pourquoi c'est un problème dans votre contexte spécifique
  • Quel risque cela représente concrètement (pas de jargon)

  • Bonnes pratiques générales liées à ce type d'issue

Suggestion de descriptions (no_alias / no_description)

L'IA analyse le YAML de votre automatisation et génère une description pertinente. Exemple :

yaml 
# Automatisation sans description :
trigger:
  - platform: state
    entity_id: binary_sensor.porte_entree
    to: "on"
action:
  - service: light.turn_on
    target: { entity_id: light.couloir }

Backend Python — Patterns HA

Toujours async

python 
# ❌ Bloquant — ne jamais faire
def load_yaml(path):
    with open(path) as f:
        return yaml.safe_load(f)

# ✅ Non-bloquant — via executor
async def async_load_yaml(hass, path):
    return await hass.async_add_executor_job(
        lambda: yaml.safe_load(open(path))
    )

Frontend JS — Règles critiques

Shadow DOM — toujours appender au body

javascript 
// ❌ Invisible avec Shadow DOM (Light DOM non rendu)
this.appendChild(modal);

// ✅ Toujours visible quel que soit le contexte Shadow DOM
document.body.appendChild(modal);

Event Delegation obligatoire

javascript 
// ❌ Fragile — listener perdu après innerHTML =
container.querySelectorAll('.btn').forEach(btn =>
    btn.addEventListener('click', handler));

// ✅ Robuste — un seul listener stable sur le conteneur
if (!container._delegated) {
    container._delegated = true;
    container.addEventListener('click', e => {
        const btn = e.target.closest('.btn');
        if (btn) handler(btn);
    });
}
